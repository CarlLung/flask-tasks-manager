Features Spotlight

Form Validations Examples:

  • 1) Used RegEx for adding constraints to registration form on username and password (Add new user).
  • 2) User cannot enter Due date that is prior to today ( Add new task).
  • 3) Cannot assign new task to users that is not found in the system ( Add new task).
  • 4) Return error if password does not match with confirm password, or credentials not matched in user.txt. (Log in and add new user).

Et cetera, and many more to be explored ...

Authorisation

  • 1) Entering links for the specific pages would be redirected to home page with error message if not logged in.
  • 2) Login session would be timed out after 5 minutes. If the user clicked any where after the time out, the user would be redirected to the login page.

Authentication

  • 1) For user - admin, the tab Statistics would appear on the side menu, but not visible to other users.
  • 2) New user registration tab is visible for all, but the form is only accessible to the admin. Other users are blocked from seeing the form and redirected to another page.

Error Handling

  • Error or success message would be returned to all the actions mentioned above.
